## 二、P0 🔴 上线阻塞（必须优先完成）

### P0-1 定价种子数据补全

**背景：** `region_pricing` 表完全没有数据，`grid_product` 缺少机构套餐产品。现在跑通创建订单流程会直接报错。

**工作内容：**
- 为所有 7 个产品（PLUS、VOCAB、GRAMMAR、CHARACTER、CONFUSING_WORDS、CULTURE、TOPIC）在 5 个区域（A/B/C/D/E）按 MONTHLY/QUARTERLY/YEARLY 插入定价数据
- A 区需同时支持 USD 和 EUR 两个币种
- 注册 3 个机构套餐产品（INST_STARTER / INST_BASIC / INST_PRO）到 `grid_product`，并插入对应区域定价
- 按需求文档定价表逐条核对（$29.99/年、¥169/年 等）

**参考文档：** §2.2 表、§2.3 表、§3.2-3.4 表
**涉及文件：** `sql/billing.sql` 种子数据
**预估工时：** 2 天

### P0-3 机构成员管理

**背景：** 机构管理员无法管理成员。`grid_organization.current_members` 永远为 1。机构产品无法运营。

**工作内容：**

| 子项 | 说明 |
|------|------|
| 成员列表 | 分页查询机构所有成员（关联 `grid_user`），展示邮箱、角色、状态、加入时间 |
| 邀请成员 | 通过邮箱邀请新成员（创建 `grid_user` + 关联机构），或拉取已有用户加入 |
| 移除成员 | 软删除成员关系，冻结账号或解除机构关联 |
| 更换成员 | 替换机制：移除 A 成员 → 添加 B 成员，记录更换次数 |
| 人数校验 | 添加/邀请时校验 `current_members < max_members`；试用期 `current_members < 20` |

**涉及文件（新增/修改）：**
- `grid-app/.../service/OrganizationMemberService.java` + impl
- `grid-app/.../rest/OrganizationMemberController.java`
- `grid-system` 或 `grid-bootstrap` 管理后台审批扩展

**参考文档：** §3.1(122行)、§3.5(165-171行)
**预估工时：** 4 天

### P0-4 试用期限制

**背景：** 试用用户当前无任何使用限制，需求文档要求在试用期间禁止批量下载/导出等功能。

**工作内容：**
- `EntitlementEngine` 新增 `isTrialUser(userId)` 方法（查询是否有 ACTIVE 且 sourceType=TRIAL 的权益）
- 批量下载/导出 API 加 `@RequireProduct` 或自定义切面，在试用期拒绝
- 防重复试用：用户在注册/发放试用前检查是否已有过 TRIAL 来源的权益记录
- 需注意：防重复的粒度 — 同一邮箱/同一设备/同一支付账户

**涉及修改：**
- `grid-billing/.../service/impl/EntitlementEngineImpl.java`
- 批量导出相关的 Controller + Aspect

**参考文档：** §2.1(47-49行)
**预估工时：** 2 天

### P0-5 推荐体系完善

**背景：** 第一期只实现了普通用户推荐（+30天PLUS）和代理商推荐（记录），缺少 6 种合作类型和完整结算流程。

**工作内容：**

**子项 A — 合作类型扩展：**
- 在 `referral_record.referrer_type` 中增加类型：

| 类型 | 奖励比例 | 奖励方式 |
|------|---------|---------|
| `CONTENT_PROMOTER` | 15% | CASH |
| `INST_LEAD` | 10% | CASH |
| `INST_FACILITATOR` | 15% | CASH |
| `REGIONAL_PARTNER` | 20% | CASH |
| `ENTERPRISE_LEAD` | 5% | CASH |
| `ENTERPRISE_FACILITATOR` | 10% | CASH |

**子项 B — 结算引擎：**
- 实现按"平台实际到账收入"计算奖励：`订单金额 × (1 - 退款率) × (1 - 税率) × (1 - 支付通道费率) × 奖励比例`
- 结算状态机完整流程：`PENDING → SETTLED → PAID`
- 后台定时任务（或手动触发）按月/季结算
- 有订单关联的推荐记录，在订单过退款期后自动结算

**子项 C — 提现功能：**
- 提现申请 API（最低 ¥1000 或等值外币）
- 提现审核后台
- 身份/税务信息收集

**涉及文件（新增/修改）：**
- `grid-app/.../service/impl/ReferralServiceImpl.java`（大幅改造）
- `grid-app/.../rest/WithdrawalController.java`（新增）
- `grid-system` 或 `grid-bootstrap` 结算管理后台

**参考文档：** §5 全部
**预估工时：** 5 天

### P0-6 机构试用期人数上限强制

**背景：** 机构审核通过后获得 30 天试用，需求要求试用期内最多 20 人，当前未做限制。

**工作内容：**

- 成员添加时校验：试用期内 `current_members < 20`

**参考文档：** §3.1(122行)
**预估工时：** 1 天

### P0-7 成员账户更换规则

**背景：** 需求要求每个账户每年可免费更换 2 次，超出收费。当前无此逻辑。

- 超额更换的收费逻辑（可复用订单体系）

**参考文档：** §3.5(167行)
**预估工时：** 2 天
