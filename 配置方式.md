# 配置方式

## 概述

本项目使用 `.env` 文件来管理不同环境的敏感配置，避免将数据库密码、API 密钥等敏感信息提交到 Git 仓库。

## 环境变量配置

### 配置文件说明

- `.env.dev` - 开发环境配置（本地开发使用）
- `.env.prod` - 生产环境配置（服务器部署使用）
- `.env` - 通用配置文件（当没有找到对应 profile 的文件时使用）
- `.env.example` - 配置模板文件（提交到 Git）
- `.env.*.example` - 各环境的配置模板（提交到 Git）

### Git 忽略规则

以下文件不会被提交到 Git：
- `.env`
- `.env.dev`
- `.env.prod`
- `.env.local`
- `.env.*`

## 快速开始

### 1. 复制配置模板

根据你的环境复制对应的配置文件：

```bash
# 开发环境
cp .env.example .env.dev

# 生产环境
cp .env.example .env.prod
```

### 2. 修改配置项

编辑对应的 `.env` 文件，填写你的实际配置：

```bash
# 开发环境
vi .env.dev

# 生产环境
vi .env.prod
```

### 3. 启动应用

#### 开发环境（默认）

```bash
# Maven 方式
mvn spring-boot:run

# JAR 方式
java -jar app.jar
```

自动加载 `.env.dev` 文件。

#### 生产环境

```bash
# 方式 1：命令行参数
java -jar app.jar --spring.profiles.active=prod

# 方式 2：系统属性
java -Dspring.profiles.active=prod -jar app.jar
```

自动加载 `.env.prod` 文件。

#### 指定其他环境

```bash
# 加载 .env.test
java -jar app.jar --spring.profiles.active=test
```

## 配置加载顺序

应用启动时会按以下顺序查找配置文件：

1. 先检查系统属性 `spring.profiles.active`
2. 再检查命令行参数 `--spring.profiles.active=`
3. 如果都没有，默认使用 `dev`
4. 尝试加载 `.env.{profile}` 文件
5. 如果不存在，尝试加载 `.env` 文件
6. 如果都没有，使用 YML 中的默认值

## 部署流程

### 开发环境部署

1. 确保 `.env.dev` 文件存在并配置正确
2. 直接启动应用即可

### 生产环境部署

#### 方式 1：使用 .env.prod 文件

1. 将 `.env.prod` 文件上传到服务器
2. 确保文件与 jar 包在同一目录
3. 启动应用：

```bash
java -jar app.jar --spring.profiles.active=prod
```

#### 方式 2：使用系统环境变量（推荐）

直接在服务器上设置环境变量，不需要 `.env` 文件：

```bash
# Docker 方式
docker run -e DB_HOST=localhost -e DB_PWD=secret ... app

# K8s 方式
env:
  - name: DB_HOST
    value: localhost
  - name: DB_PWD
    valueFrom:
      secretKeyRef:
        name: db-secret
        key: password

# 云服务商控制台配置
# 在云服务商的控制台中直接设置环境变量
```

## 开发说明

### 新增配置项

如果需要新增配置项，需要按以下步骤：

1. 在 YML 配置文件中使用 `${ENV_VAR:default}` 格式
2. 在 `.env.example` 中添加配置项说明
3. 在各环境的 `.env.*` 文件中添加实际值

### 修改现有配置

1. 优先修改 `.env` 文件
2. 如果需要修改默认值，再修改 YML 配置

### 验证配置

启动应用时会打印当前加载的配置文件：

```
Loaded config from .env.dev
```

或

```
Loaded config from .env
```

或

```
No .env file found, using default configs
```

## 安全建议

1. **永远不要**将 `.env`、`.env.dev`、`.env.prod` 等包含真实密码的文件提交到 Git
2. 生产环境建议使用系统环境变量或密钥管理服务，不使用 `.env` 文件
3. 定期更换数据库密码、JWT 密钥等敏感信息
4. 设置 `.env` 文件权限为仅当前用户可读取：`chmod 600 .env`

## 回退方案

如果 dotenv-java 不适合项目，可以：

1. 移除 pom.xml 中的 dotenv-java 依赖
2. 恢复 AppRun.java 中的原始代码
3. 直接在 application-dev.yml 和 application-prod.yml 中配置
4. 或者使用 Spring Cloud Config 等配置中心

## 常见问题

### Q: 配置不生效怎么办？

A: 检查以下几点：
1. 确保 `.env` 文件与 jar 包在同一目录
2. 检查 profile 是否正确指定
3. 查看启动日志中的配置加载提示

### Q: 如何在 IDE 中配置？

A: 在 IDE 的运行配置中：
1. 可以设置 VM options: `-Dspring.profiles.active=dev`
2. 或者直接设置 Environment variables
3. 或者将 `.env` 文件放在项目根目录

### Q: 支持的 Spring Boot 版本？

A: Spring Boot 2.x 和 3.x 都支持

### Q: 配置项名称有什么要求？

A: 使用大写字母和下划线，如 `DB_HOST`，不要使用短横线或点号
