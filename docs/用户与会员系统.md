# APP 用户与会员系统接入指南

## 概述

APP 端用户体系包含两大部分：
- **用户认证**：注册、登录、Token 刷新、退出（详见 `docs/登录与鉴权.md`）
- **会员订阅**：基于 `grid_user_role` 表 + `@RequireSubscription` 注解的鉴权体系

---

## 一、用户角色体系

### 数据库模型

用户拥有的角色/会员信息存储在 `grid_user_role` 表：

| 字段 | 说明 |
|------|------|
| `user_id` | 用户 ID |
| `role_code` | 角色编码：`NORMAL` / `VIP` / `SVIP` |
| `role_name` | 角色名称：普通用户 / VIP会员 / SVIP会员 |
| `expire_time` | 过期时间（`NULL` 表示永久有效，仅 `NORMAL` 用） |

**用户角色组合规则：**

| 用户类型 | 持有的角色记录 | 说明 |
|---------|---------------|------|
| 普通用户 | `NORMAL` | 注册后自动获得，永久有效 |
| VIP 会员 | `NORMAL` + `VIP` | `VIP` 带 `expire_time` |
| SVIP 会员 | `NORMAL` + `SVIP` | `SVIP` 带 `expire_time` |
| VIP → SVIP 升级 | `NORMAL` + `VIP` + `SVIP` | 旧 VIP 保留不动，不折算 |

**关键约束：**
- `(user_id, role_code)` 唯一索引，同一用户同一角色只有一条记录
- VIP 和 SVIP 是平级关系，SVIP 用户**不**自动持有 VIP 角色
- SVIP 能访问 VIP 内容是通过 `UserLevel.includes()` 层级逻辑实现的（见下文）

## 二、`@RequireSubscription` 注解

### 注解说明

在 Controller 方法上标注 `@RequireSubscription`，即可对该接口做会员鉴权。

```java
@RequireSubscription(UserLevel.VIP)   // 需要 VIP 或更高级别
@RequireSubscription(UserLevel.SVIP)  // 仅 SVIP 可访问
@RequireSubscription()                // 默认 = UserLevel.VIP
```

### 鉴权流程（自动执行，无需额外操作）

```
请求到达 → SubscriptionAspect 拦截
  ├─ 1. 从 SecurityContext 获取 AppAuthenticationToken
  ├─ 2. 从 JWT roles 中快速检查是否有 VIP/SVIP
  │    ├─ 无 → 返回 400 "需要订阅后才能访问此内容"
  │    └─ 有 → 继续
  ├─ 3. 查数据库验证 expire_time 是否有效
  │    ├─ 已过期 → 返回 400 "订阅已过期，请续费"
  │    └─ 有效 → 继续
  ├─ 4. 比较用户级别 vs 注解要求的级别
  │    ├─ 用户级别不够 → 返回 400 "需要订阅后才能访问此内容"
  │    └─ 满足 → 放行
  └─ 5. 执行原始方法
```

### 级别层级

`UserLevel` 枚举定义了层级关系：

```java
public enum UserLevel {
    VIP(1),   // 基础会员
    SVIP(2);  // 高级会员（包含 VIP 全部权限）

    public boolean includes(UserLevel other) {
        return this.level >= other.level;
    }
}
```

| 用户级别 | 可访问 `@RequireSubscription(VIP)` | 可访问 `@RequireSubscription(SVIP)` |
|---------|-----------------------------------|-----------------------------------|
| `NORMAL` | ❌ | ❌ |
| `VIP` | ✅ | ❌ |
| `SVIP` | ✅ | ✅ |

### 使用示例

```java
@RestController
@RequestMapping("/api/app/vocab")
public class AppVocabWordController {

    // 所有用户可访问（不用注解）
    @GetMapping("/search")
    public ResponseEntity<> search() { ... }

    // VIP 及以上可访问
    @RequireSubscription(UserLevel.VIP)
    @GetMapping("/{id}")
    public ResponseEntity<> getDetail(@PathVariable Integer id) { ... }

    // 仅 SVIP 可访问
    @RequireSubscription(UserLevel.SVIP)
    @GetMapping("/advanced-analysis")
    public ResponseEntity<> getAdvancedAnalysis() { ... }
}
```

### 与 `@AnonymousGetMapping` 的区别

| 注解 | 效果 |
|------|------|
| `@AnonymousGetMapping` | 无需任何认证，公开可访问 |
| `@GetMapping`（无注解） | 需要登录，但不检查会员等级 |
| `@RequireSubscription(VIP)` | 需要登录 + VIP 或 SVIP 会员 |
| `@RequireSubscription(SVIP)` | 需要登录 + SVIP 会员 |

### 注意事项

1. **JWT 角色缓存**：用户登录时，当前角色会写入 JWT。购买订阅后，用户需要**重新登录**或**刷新 Token** 才能获得更新后的角色信息。在此期间，即使数据库中已有 VIP 角色，JWT 中可能还没有，Aspect 会拒绝访问。
2. **注解必须标注在公开的 Controller 方法上**（被 Spring 管理的 Bean）。
3. **不要将 `@RequireSubscription` 和 `@AnonymousGetMapping` 同时使用**——后者会让接口完全公开，会员检查不会生效。

## 三、会员订阅 API

### 3.1 查询我的订阅状态

```
GET /api/app/subscription/my
Authorization: Bearer {accessToken}
```

**响应：**
```json
{
  "level": "VIP",
  "expireTime": "2026-07-16T15:00:00.000+00:00",
  "expiringSoon": false
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `level` | String | `NORMAL` / `VIP` / `SVIP` |
| `expireTime` | String (ISO8601) 或 `null` | 订阅到期时间，`null` 表示未订阅 |
| `expiringSoon` | boolean | 是否 15 天内到期 |

该接口**不需要会员鉴权**，所有已登录用户都可以调用。

### 3.2 创建订阅订单（预留）

```
POST /api/app/subscription/create-order
Authorization: Bearer {accessToken}
Content-Type: application/json

{
  "level": "VIP",
  "periodMonths": 1
}
```

**V1 仅做参数校验和占位**，返回 mock 的 orderId。实际支付集成（StoreKit / Google Play / 支付宝等）需后续对接。

### 3.3 激活订阅（内部接口）

支付系统回调时调用 `SubscriptionService.activateSubscription()`，写入或续期用户角色。该接口**不在 Controller 层暴露**，仅通过 Service 层内部调用。

## 四、试用机制

### 注册自动试用

所有新用户注册时自动获得 **7 天 VIP 试用**（天数可配置，通过设置 `app.subscription.trial-days` 为 `0` 可关闭）：

```yaml
# application.yml
app:
  subscription:
    trial-days: 7    # 设为 0 关闭自动试用
```

试用到期后，`grid_user_role` 中的 `VIP` 记录保留但标记为过期，`SubscriptionAspect` 会返回 "订阅已过期" 错误。

### 试用规则

- 每个用户**仅限一次**试用（`grid_user_role` 的 `uk_user_role` 唯一约束保证）
- 试用的是 **VIP** 级别（不是 SVIP）
- 试用到期后**不会自动扣费续期**（支付未接入）

## 五、完整接入时序

### 用户注册 → 试用 → 购买 VIP

```
前端                                 服务端
  │                                    │
  │  POST /api/app/auth/register       │
  │ ──────────────────────────────────→ │  创建用户 + NORMAL 角色
  │                                    │  创建 VIP 试用角色（7天）
  │                                    │  生成 JWT（roles: [NORMAL, VIP]）
  │  ←─────────── TokenDTO ──────────── │
  │                                    │
  │  GET /api/app/vocab/search          │  ← 公开接口，无需鉴权
  │ ──────────────────────────────────→ │
  │                                    │
  │  GET /api/app/vocab/1              │  ← 标注了 @RequireSubscription(VIP)
  │ ──────────────────────────────────→ │  JWT 中有 VIP → DB 检查未过期 → 放行
  │                                    │
  │  ...7天后试用到期...                │
  │                                    │
  │  GET /api/app/vocab/1              │
  │ ──────────────────────────────────→ │  JWT 中有 VIP（旧）→ DB 检查已过期 → 拒绝
  │  ←── 400 "订阅已过期，请续费" ──── │
  │                                    │
  │  (用户购买，支付系统回调)             │
  │  subscriptionService.activate()    │
  │  （内部调用）                        │  更新 VIP 的 expire_time 延长
  │                                    │
  │  用户刷新 Token                      │
  │  POST /api/app/auth/refresh         │
  │ ──────────────────────────────────→ │  新 JWT（roles: [NORMAL, VIP] 新到期时间）
  │  ←─────────── TokenDTO ──────────── │
  │                                    │
  │  GET /api/app/vocab/1              │
  │ ──────────────────────────────────→ │  恢复访问
```

## 七、错误码

| HTTP 状态码 | 业务含义 | 说明 |
|------------|---------|------|
| 400 | 需要订阅后才能访问此内容 | 用户未订阅或级别不足 |
| 400 | 订阅已过期，请续费 | 用户的 VIP/SVIP 已到期 |
