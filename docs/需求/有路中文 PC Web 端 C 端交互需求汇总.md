# 有路中文 PC Web 端 C 端交互需求汇总

## 说明

- 每个板块统一按"学习者场景、关键交互、数据资源、待定事项、跨板块关系"呈现，便于产品、内容、技术和设计协同查看。
- 文末单独汇总优先级与待办（仅供参考），便于逐项确认。
- 本期优先建设 PC Web 端，主要运行于桌面浏览器，并兼顾基础响应式适配。本期不开发原生 iOS 或 Android App。
- 本期需求设计应以 PC 浏览器中的搜索、资源查看、跨板块跳转、账号试用、权限控制、会员购买、收藏和个人中心闭环为基础。

| 全局搜索 | 词汇、汉字、语法、功能表达、易混淆词辨析 | 输入一个词、字或表达需求，快速找到相关资源 | 搜索建议、分类结果、跨板块结果、新标签页打开 | |
| 账号与会员 | 14 天限时体验、每日 3 个完整条目、单模块订阅、Plus 会员 | 注册、查看体验额度、购买模块或 Plus | 登录注册、剩余额度、付费墙、套餐比较、订单与会员状态 | |

### 一期重点建议

放在：**全局搜索、词汇详情页、汉字详情页、语法详情页、功能句式集、每日一语卡片**，以及**跨板块跳转与个人收藏**这两条共通能力，以及**账号注册、7 天限时体验、单模块权限、Plus 会员权限和购买流程**。

话题场景、词语网络、文化板块等内容暂不进入 1.0 版本。

## 二、跨板块联动关系

跨板块跳转应作为独立能力设计和验收。建议在数据层先建立稳定 ID 与关系表，再在前端呈现为自然的入口，而不是等各板块完成后再临时补链接。

| 入口节点 | 应能跳转到 | 关键数据关系 | 验收要点 |
|---------|-----------|-------------|---------|
| 词条 | 组成它的单字；相关语法点；相关功能表达；近义、反义、正序、逆序、其他（词汇关联类型已注明）。 | word_id ↔ char_id；word_id ↔ grammar_id；word_id ↔ function_id；word_id ↔ related_word_id。 | 从词条进入关联资源后，能回跳到原词；同一资源不出现多份不一致内容。 |
| 汉字 | 部首-同部首其他汉字、组词、词条详情。 | char_id ↔ radical_id；char_id ↔ word_id。 | 从字进入组词时，除了"汉字数据表"中的简易组词信息，还可以实现调用词库同一条词条记录，获得完整词条内容。 |
| 语法点 | 词条、功能句式。 | grammar_id ↔ word_id；grammar_id ↔ function_pattern_id。 | 语法例词可点；功能句式对应语法点时可跳转。 |
| 功能句式 | 相关语法点、相关词条。 | function_id ↔ grammar_id；pattern_id ↔ word_id。 | 句式差异、语气和场合标注清晰；可收藏。 |
| 每日一语 | 词汇板块词条；个人收藏；历史归档。 | daily_id ↔ word_id；daily_id ↔ collection_id。 | 轻量内容可继续深入；曾推送内容可回查。 |
| 个人学习空间 | 词、字、语法点、功能句式、每日一语。 | collection_id 统一管理不同资源类型。 | 收藏入口和分类逻辑一致；可按主题、教材或自定义标签分组。 |

### 扩充"个人学习空间"

原来主要是收藏功能，应改为：

| 功能 | PC Web 端内容 |
|------|--------------|
| 我的收藏 | 按词汇、汉字、语法、功能、每日一语分类 |
| 最近浏览 | 显示近期查看过的资源 |
| 自定义收藏夹 | 支持主题、教材、课程或自定义标签 |
| 试用状态 | 显示剩余试用天数和今日剩余完整条目数 |
| 会员状态 | 显示已购买模块、Plus 状态和到期时间 |
| 订单记录 | 显示购买记录、支付状态、发票或收据入口 |
| 升级入口 | 从单模块升级到 Plus 并显示可抵扣金额 |

### 状态标签说明

- **可落地**：现有字段或现有规则基本可以支撑前端实现。
- **需补**：内容、字段或资源仍需补齐。
- **待定**：需要产品、内容、技术或设计先做决策。
- **风险**：如果不先解决，会影响内容一致性、交互闭环或上线验收。
- **跨板块**：涉及多个板块的 ID、跳转、收藏或展示规范。

## 三、词汇板块

### 板块定位

- 词汇是平台体量最大、学习者使用最频繁的板块，约 1.5 万词条。
- 现有词汇数据集已经支持"义项-释义-例句-搭配-关系词-多模态"的丰富详情页。
- 词条应成为平台跳转网络的中心节点。

### 3.1 使用场景

| 学习者动作 | 典型场景 | 核心需求 | 界面侧重 |
|-----------|---------|---------|---------|
| 带着词来查 | 在课文、对话、字幕或练习中遇到生词。 | 快速知道意思、读音和具体用法。 | 详情页要查得快、定位准，义项和例句清楚。 |
| 系统地学 | 按 HSK 等级、话题或备考目标刷词。 | 批量浏览、筛选、收藏，形成个人词单。 | 列表页与筛选页要好用，词单管理要顺。 |

### 3.2 关键交互与页面模块

| 模块 | 整理后的交互呈现 | 数据支撑 / 备注 |
|------|-----------------|----------------|
| 词条详情页 | 按义项分块呈现。一个词的多个义项分别显示词性、中英释义、配图和例句，避免把释义笼统堆在一起。设置折叠界面（大于 1 个义项的词条都折叠），折叠后只保留词性、拼音、释义，确保界面更直观。 | 核心原则是"义项级颗粒度"。之前讨论过的"花""安心"等例子都体现了词位分立的思路。 |
| 释义区 | 每个义项包含 HSK 等级、词性标签、中文释义、外文释义和释义配图。 | 具体名词、动作动词可配图；抽象词和虚词不强行配图，改用释义提示。 |
| 例句区 | 每个义项挂若干条例句，例句带拼音、翻译、配图（并非所有例句都配图）和朗读；被讲解词高亮。 | 例句本身在词汇详情中的承载位置需要内容侧和产品侧确认。 |
| 搭配与结构区 | 展示典型构式和结构搭配，例如"花 + O[时间/钱/功夫/精力]"。 | A 表中的"结构搭配"字段可直接支撑。 |
| 易混淆词辨析 | 词条详情页提供"易混淆词辨析"入口，一键跳转到该词与易混词的辨析页。辨析页把两个词放在一起对比（交互式界面，参考飞象老师 demo），辨析页底部挂配套练习，一期统一为选择题（选词填空形式），即时反馈加解析。 | 数据已基本就绪，易混淆词 demo 含辨析头、词头、词性、用法对比（含翻译）、例句、通用用法、练习题等字段。需要交互式处理。 |
| 词语关系区 | 近义、反义、正序、逆序、其他五类关系做成可点击入口。点击后跳转到对应词条详情页，是词汇内部互联的基础。 | — |
| 发音与收藏 | 词和例句均支持标准朗读；词条均可一键收藏。 | 音频统一 AI 合成，仅中文；收藏进入个人词单。 |

### 3.3 同形词呈现（需要专门处理）

| 同形词类型 | 例子 | 呈现建议 |
|-----------|------|---------|
| 同形异音异义 | 行 xíng / háng | 检索结果按字形聚合，但优先按读音分流（设置为检索并列呈现）；读音不同的词，对学习者基本就是两个词。 |
| 同形同音异义 | 花：花钱 / 鲜花 | 详情页在词头下方设置平级胶囊标签（点击切换），标签上直接标明词性、读音、序号。 |
| 同形同音但意义分化为分立词条 | 安心：存心 / 心安 | 数据层继续保持分立；呈现层做智能聚合，避免检索结果零散。 |

**处理建议**：检索结果先按字形聚合，再用读音和 HSK 等级排序，高频常用词排前；详情页将 W1/W2 类切换从底部挪到词头正下方。

### 3.4 词汇游戏：词汇大挑战

| 出题规则 | 交互要求 | 内容备注 |
|---------|---------|---------|
| 题目从总词库实时调取；每题包含词汇、等级、配图、文案、候选词汇五类字段。 | 前端完成选择、反馈、计分和错题记录。 | 具体名词、动词用图片提示；抽象词改用英文释义提示。 |

### 3.5 需求点与待定事项

| 状态 | 整理后的需求点 | 对齐对象 / 备注 |
|------|---------------|----------------|
| ✅ 可落地 | 义项分块详情页、搭配结构呈现、词语关系跳转、例句多模态、词汇大挑战出题逻辑。 | 产品可先出详情页与列表页原型。 |
| ⏳ 需补 | AI 生成释义配图与例句配图字段。 | 内容+技术确认字段和资源上传方式。 |
| ❓ 待定 | 同形词检索排序与详情切换交互；抽象词/虚词不配图的判定规则。 | 建议"按词性自动判定 + 人工可覆盖"。 |
| 🔗 跨板块 | 词↔字、词↔语法点、词↔功能表达的双向跳转入口。 | 需要统一 ID 与关系表。 |
| ⏳ 需补 | 全局搜索结果中的词汇聚合、同形词排序和快速预览。 | PC 首页和搜索结果页的核心能力。 |
| ⏳ 需补 | 词汇模块权限控制与付费墙。 | 区分试用、词汇模块会员、Plus 会员和未购买状态。未购买词汇模块或 Plus 会员的用户，在试用期内按每日免费额度查看完整词条；额度用尽或试用结束后，仅显示词头、拼音、简要释义和会员购买入口。 |

## 四、汉字板块

### 板块定位

- 汉字板块覆盖约 3000 常用字和 201 个部首，其中约 100 个为核心部首。
- 不只是查字工具，还要承担"写"和"理解结构"的学习功能。
- 汉字与词汇的关联必须先理顺，否则字、词两个入口看到的内容可能不一致。

### 4.1 使用场景

| 学习者动作 | 典型问题 | 界面侧重 |
|-----------|---------|---------|
| 查单字 | 这个字怎么读、是什么意思、怎么写、能组什么词。 | 字头详情、拼音、简繁、部首、部件组合、组词和例句。 |
| 系统学字 | 按部首归类记忆，按字源理解造字理据。 | 部首-字族浏览、字源说明、形近字辨析。 |
| 练习书写 | 需要字帖、笔顺和田字格练习。 | 字帖 PDF 生成、笔顺示意、描摹和空白练习页。 |

### 4.2 关键交互与页面模块

| 模块 | 整理后的交互呈现 | 数据支撑 / 备注 |
|------|-----------------|----------------|
| 字头详情 | 显示字形（简/繁）、拼音（多音字分列）、HSK 等级、部首、部件组合（如"车+俞=输"）、字源相近的关联部首（如"心"和"忄"）。 | demo 字段已较完整，可直接支撑详情页。 |
| 字源说明 | 用中英文说明造字法、字形来源、本义和引申义等信息。 | 建议配字源演变图：具体形象→汉字（例如"山""水"）；风格可偏水墨。 |
| 形近字辨析 | 列出易混字，说明共享部件和意义差别，例如"输 vs 愉"。 | demo 的"辨析字"字段已包含拼音、英文释义和对比说明。 |
| 组词与例句 | 每个字挂一组词语，词语显示等级、拼音、词性、英文释义、例句、例句图片。点击组词应跳到词汇板块同一条词条记录。 | — |
| 部首-字-词-句链路 | 从部首进入可看到字族；从字进入可看到组词、例句、字源相近的关联部首（如"心"和"忄"）。 | 宀部 demo 已展示"部首→字→词→句"的组织方式。 |

### 4.3 字帖生成：突出"写"的能力

| 功能 | 学习者端 | 教师端 | 资源依赖 |
|------|---------|-------|---------|
| 字帖生成 | 可选单字、字族或自定义字单，生成 PDF 字帖、下载 PDF 字帖、浏览器打印。 | 可批量生成某等级或某教材的字帖作为作业。 | 需要笔顺、笔画、田字格模板和笔顺动画资源（SVG/Lottie）。 |
| 字帖内容 | 笔顺示意、田字格描摹、空白练习页。 | 可按课程或作业范围批量导出。 | 需先确认"笔顺"字段到底是笔画数、笔顺序列，还是动画引用。 |

### 4.4 数据结构隐患：组词与词库必须打通

需要优先处理的风险：

- 汉字 demo 中的组词目前像宽表一样铺在字这一行里，组词的拼音、释义和例句在字下重新录入。
- 同一个词可能在汉字库和词汇库各存一份，导致"从字进入"和"从词汇板块查到"的内容不一致。可以先呈现汉字库的简要内容，然后点击汉字进入词汇库详情界面，实现由字到词的衔接。
- **建议**把组词关联到词库的同一条 word_id。C 端无论从哪个入口进入，都调用同一份权威词条内容。

### 4.5 汉字游戏

| 游戏类型 | 可用字段 | 出题思路 |
|---------|---------|---------|
| 典型部首识别 | 部首字段、核心部首库。 | 给字或部件，让学习者选部首。 |
| 形近字辨析 | 辨析字字段、对比说明。 | 给语境或释义，让学习者区分易混字。 |
| 汉字组词 | 组词字段、词库关联。 | 给字选词、给词找字，或根据例句补词。 |

### 4.6 需求点与待定事项

| 状态 | 整理后的需求点 | 对齐对象 / 备注 |
|------|---------------|----------------|
| ✅ 可落地 | 字头详情、字源说明、形近字辨析、部首-字族浏览、组词跳转、三类汉字游戏。 | 可先做详情页与部首入口原型。 |
| ⏳ 需补 | 笔顺动画（SVG/Lottie）、字源演变图序、字帖 PDF 生成模板。 | 内容+设计+技术共同确认资源规格。 |
| ❓ 待定 | "笔顺"字段含义；多源部首的对比展示方式，例如月部来自"月亮"和"肉"的不同来源。 | 内容侧先定义，技术侧再落库。 |
| ⚠️ 风险 | 组词与词库未关联，存在重复存储与内容不一致风险。 | 实现链接跳转。 |
| ⏳ 需补 | 汉字模块的试用额度、单模块购买和 Plus 权限控制。 | 未购买汉字模块或 Plus 会员时，在试用期内按每日免费额度查看完整信息；额度用尽或试用结束后，仅显示字头、拼音、部首和少量组词预览和会员购买入口。 |

### 5.1 使用场景

| 按等级系统复习 | 按 HSK 等级过语法清单，服务备考。 | 列表筛选、分类体系、选择题练习、错题记录。 |

## 六、功能板块

### 板块定位

- 1.0 版本暂不做"话题"，只做"功能"。
- 功能板块按交际目的组织语言，解决"这个意思，中文怎么说"的问题。
- 它强调可对比、可挑选、可跟读，而不是单向展示一个固定答案。

### 6.1 使用场景与交互感

| 学习者需求 | 板块回应方式 | 交互重点 |
|-----------|-------------|---------|
| 想表达某个交际目的。 | 围绕一个功能给出一组可选句式。 | 让学习者看到不同说法的语气、场合和常用程度。 |
| 想说得更自然。 | 同一功能下提供细微差别提示。 | 帮助学习者选对表达，而不是只记一个句子。 |
| 想练口语。 | 句式和例句支持朗读与跟读。 | 强化"能说出来"。 |

### 6.2 关键交互与页面模块

| 模块 | 整理后的交互呈现 | 数据字段建议 |
|------|-----------------|-------------|
| 句式集呈现 | 一个功能下的多个句式并列展示，每条配例句、翻译和音频。 | 功能名、句式、例句、翻译、音频。 |
| 对比与挑选 | 给出同功能句式之间的细微差别，例如口语/正式、随意/不耐烦。 | 语气、场合、常用度、适用对象。 |
| 跟读练习 | 句式和例句均可播放标准朗读，提供标准音频播放、暂停、重播和速度调整。 | 音频资源；后续可接语音评测。 |
| 收藏 | 可收藏单个句式，也可收藏整组功能。 | collection_id 统一收藏。 |

### 6.3 数据结构建议

功能项的数据结构可以先保持轻量：一个功能名下挂一组句式，每个句式带例句、翻译、音频、语气/场合标注。此时不必提前建完整话题关联。后续做话题时，再把功能挂接到相关话题下，同一功能可被多个话题复用。

| 建议表 | 核心字段 | 说明 |
|-------|---------|------|
| 功能表 | function_id、功能名称、功能说明、等级、排序。 | 承载"表达听任"等功能项。 |
| 功能—句式表 | pattern_id、function_id、句式、例句、翻译、音频、语气/场合、常用度、关联语法点。 | 一期最关键，需内容侧先定义。 |
| 收藏关联表 | collection_id、user_id、resource_type、resource_id。 | 实现单句式和整组功能收藏。 |

### 6.4 需求点与待定事项

| 状态 | 整理后的需求点 | 对齐对象 / 备注 |
|------|---------------|----------------|
| 📌 一期范围 | 仅做功能板块；话题（购物、看病等场景包）延后。 | 话题涉及词群、对话、分级文本等大量资源。 |
| ✅ 可落地 | 功能句式集呈现、句式对比挑选、跟读、收藏。 | 可先用 demo 内容验证交互。 |
| 📊 数据待备 | "功能—句式"表结构尚未定义；句式的语气/场合标注字段需补。 | 内容侧先定字段。 |
| ❓ 待定 | 功能项的颗粒度与命名；同一功能内句式排序逻辑。 | 可按常用度、语气或难度排序。 |
| 🔗 跨板块 | 句式↔语法点链接；许多功能句式本身对应某个语法点。 | 建立 function_pattern_id 与 grammar_id 关系。 |

## 八、每日一语板块

### 板块定位

- 每日一语是 PC 首页的轻量化入口，主打主动触达、趣味学习和分享传播。
- 它不是让学习者主动检索，而是在打开主页浏览时吸引学习者。
- 内容应通俗、有趣、可收藏、可分享。

### 8.1 使用场景与交互感

| 使用状态 | 设计目标 | 交互重点 |
|---------|---------|---------|
| 用户访问网站首页或登录个人账户 | 用一条有趣表达吸引用户继续看。 | 首页首屏固定卡片或右侧内容区。 |
| 用户对当天内容不感兴趣 | 降低流失，允许继续探索。 | 点击"换一换"，从当日备选池再随机一条。 |
| 用户觉得内容有用或有趣 | 沉淀学习资产并带来传播。 | 一键收藏、分享到社交媒体、历史归档。 |

### 8.2 内容与交互模块

| 模块 | 整理后的交互呈现 | 说明 |
|------|-----------------|------|
| 首页"每日一语"卡片 | 每日只推一条，在主页首屏展示。 | 推送形式待产品确定。 |
| 通俗讲解 | 包含词目、读音、通俗释义、来历/故事、当代用法例句。 | 语言避免学术腔，重点是"看得懂、记得住"。 |
| 多模态呈现 | 配图、配音；成语俗语可配 AI 情景图。 | 配图风格要统一。 |
| 换一换 | 当天设置 3-5 条备选内容，用户可手动切换。 | 避免"今天这条没兴趣"造成流失。 |
| 收藏与分享 | 一键收藏进个人空间，或分享到社交媒体。 | 分享是板块自带传播属性。 |
| 历史归档 | 可往回翻近期内容，错过的能补看。 | 可按日期、类型检索。 |

### 8.3 内容范畴与数据字段

| 内容类型 | 字段建议 | 一期数据策略 |
|---------|---------|-------------|
| 新词新语、成语、俗语、惯用语、歇后语 | 内容、类型、读音、释义、出处/故事、当代用法例句、配图、配音、展示日期、备选池标记。 | 一期先整理 1000 条结构化数据，由运营提前排期。 |

### 8.4 与其他板块的关系

每日一语中出现的词目应能跳到词汇板块的词条详情；词汇板块中的成语、俗语也可标注"曾作为每日一语推送"。它与功能板块都涉及地道表达，但分工不同：每日一语主打推送和趣味，功能板块主打按交际目的成组学习。

### 8.5 需求点与待定事项

| 状态 | 整理后的需求点 | 对齐对象 / 备注 |
|------|---------------|----------------|
| ✅ 可落地 | 主页卡片、通俗讲解卡片、换一换、收藏分享、历史归档。 | 产品可先定入口位置。 |
| 📊 数据待备 | 1000 条结构化数据；内容/类型/读音/释义/出处/配图/配音/推送日期等字段需细化。 | 内容+运营先定。 |
| ❓ 待定 | 推送形式：PC 1.0 采用首页常驻卡片；趣味讲解文案风格与配图风格基调。 | 产品+设计对齐。 |
| 🔗 跨板块 | 每日一语词目↔词汇板块词条；与功能板块在"地道表达"上的分工。 | 建立 daily_id 与 word_id 关系。 |
