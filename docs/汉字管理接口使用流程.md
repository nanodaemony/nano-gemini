# 汉字管理接口使用流程

## 接口列表

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/character` | 分页查询汉字列表（后台） |
| GET | `/api/character/{id}` | 查询汉字详情（后台） |
| POST | `/api/character` | 新增汉字 |
| PUT | `/api/character/{id}` | 修改汉字 |
| DELETE | `/api/character/{id}` | 删除汉字 |
| PUT | `/api/character/{id}/review` | 审核通过 |
| PUT | `/api/character/{id}/publish` | 发布汉字 |
| PUT | `/api/character/{id}/offline` | 下线汉字 |

---

## 使用流程

### 流程一：新建并发布汉字

```
1. POST /api/character          → 新建汉字（草稿状态）
   ↓
2. PUT /api/character/{id}      → 编辑修改（可选，可多次）
   ↓
3. PUT /api/character/{id}/review  → 审核通过
   ↓
4. PUT /api/character/{id}/publish → 发布（用户可看到）
```

### 流程二：修改已发布的汉字

```
1. PUT /api/character/{id}      → 修改已发布的汉字（自动变回草稿状态）
   ↓
2. PUT /api/character/{id}/review  → 再次审核通过
   ↓
3. PUT /api/character/{id}/publish → 再次发布
```

### 流程三：下线已发布的汉字

```
已发布的汉字
   ↓
PUT /api/character/{id}/offline → 下线（用户看不到，后台仍可见）
```

下线后如需重新发布：
```
PUT /api/character/{id}/review  → 先审核
   ↓
PUT /api/character/{id}/publish → 再发布
```

### 流程四：删除汉字

```
PUT /api/character/{id}/offline → 先下线（可选）
   ↓
DELETE /api/character/{id}      → 删除（所有人都看不到）
```

## 状态说明

### 查询列表/详情时的状态标识

- `publishStatus`: 
  - `unpublished` - 未发布
  - `published` - 已发布

- `editStatus`:
  - `draft` - 草稿
  - `reviewed` - 已审核
  - `published` - 已发布

- `hasDraft`: true/false - 是否有草稿内容

### 各状态的可用操作

| 当前状态 | 可操作 |
|---------|--------|
| 草稿 | 编辑、审核、删除 |
| 已审核 | 编辑（变回草稿）、发布、删除 |
| 已发布 | 编辑（变回草稿）、下线、删除 |
| 已下线 | 编辑、审核、发布、删除 |

---

## 前端使用建议

### 新建场景
1. 调用 `POST /api/character` 保存草稿
2. 如需修改，调用 `PUT /api/character/{id}` 多次保存
3. 确认无误后，调用 `PUT /api/character/{id}/review` 审核
4. 最后调用 `PUT /api/character/{id}/publish` 发布

### 编辑已发布内容
1. 直接调用 `PUT /api/character/{id}` 编辑（会自动变回草稿状态）
2. 后续流程同新建场景

### 下线与删除
- 若只是想让用户看不到，调用 `offline`
- 若想彻底删除（后台也看不到），调用 `delete`

### 详情展示
- 调用 `GET /api/character/{id}` 即可，接口会自动返回最新数据
